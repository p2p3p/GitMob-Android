### 5. 认证体系全面重构：GitHub App 与后端架构升级
- [ ] **多模式认证体系重构 (Client)**

  - **数据层适配**：重构持久化加密存储，增加对 `refresh_token` 和 `expires_at` 等生命周期字段的管理在，确保刷新过程中的原子性。

- [ ] **Worker 项目重构：多模式适配 (Backend)**
  - **双协议支持**：重构 Cloudflare Worker 代码，确保一套代码同时支持 OAuth App 和 GitHub App 的回调逻辑与密钥管理。
  - **落地页路由重组**：
    - 重新定义落地页路径，明确划分为 `/oauth/` 和 `/github/` 两个独立入口。
    - **API 路径映射**：所有 API 请求均在落地页路径后追加（如 `/oauth/callback` 或 `/github/callback`）。
    - **后端统一封装**：重构核心逻辑层，使一套 Worker 代码通过环境变量动态适配两套 Client ID 与 Secret 体系。

- [ ] **API 适配与交互优化**
  - **动态 BaseURL**：Retrofit 需根据当前用户选择的登录模式，动态指向对应的落地页 API 根路径。
  - **授权 UI 升级**：登录界面增加「GitHub App 认证」选项，并与新的后端落地页地址完成对接。
  - 完善授权后的撤销与重新授权流程，确保两套体系的 Token 存储互不干扰。

### 6. 日志系统深度迁移与重构
- [ ] **方案迁移：全面采用 `inline + reified` 扩展**
  - 彻底废弃现有的 `LogManager` 方案，不再使用类静态引用
  - 利用 Kotlin 特性，通过 `inline` 函数配合 `reified` 自动捕获调用类名作为 TAG
  - **同步修改**：清理全工程中所有旧版日志调用，确保无残留引用
- [ ] **日志分类精简与统一**
  - 移除多种日志分级（如 Info, Warn 等），全量统一为 `debug` 级别
  - 将原代码中散落在各处的差异化日志方法按新方案统一修改
- [ ] **设置项功能改造**
  - 在设置中新增「日志记录」开关，默认状态为**关闭**
  - **样式参考**：开关 UI 效果完全参考现有的「Root 模式」开关
  - **运行逻辑**：
    - 关闭状态：不产生、不保存任何 `debug` 日志
    - 开启状态：产生并记录 `debug` 日志
    - **崩溃日志 (Critical)**：崩溃日志不受开关状态限制，必须始终记录；导出日志功能需支持勾选导出崩溃记录
- [ ] **业务流程日志补全**
  - 在不改变现有逻辑的前提下，为以下关键路径补充日志：
    - 认证流（登录、Token 校验、授权、撤销）
    - 导航流（主页加载、页面跳转、Fragment 切换）
    - 核心业务（仓库详情初始化、各 Tab 数据加载过程）
    - 等等

### 8. 资源下载体验优化：断点续传与任务控制
- [ ] **多场景下载支持断点续传**
  - **核心范围**：覆盖 Action 运行产物 (Artifacts)、Release 发布页面中的文件附件、以及应用自身的「检查更新」下载流。
  - **技术实现**：通过 OkHttp 封装支持 HTTP `Range` 请求，本地实现文件空洞填充与偏移量校验，确保中断后无需重新开始，节省流量。
- [ ] **下载任务深度控制**
  - **暂停与恢复**：为上述所有下载场景增加「暂停」和「继续」功能，支持实时挂起任务状态。
  - **状态持久化**：支持应用重启或网络切换后，能够从本地记录的已下载位置继续任务。
- [ ] **交互与反馈增强**
  - **组件按钮**：在 Artifacts 列表、Release 文件列表及更新进度弹窗中，新增暂停/恢复切换按钮。
  - **通知栏管理**：
    - 任务执行时弹出前台通知栏，并实时显示下载进度条。
    - **通知栏交互**：在通知栏集成「暂停」、「继续」和「取消」操作按钮，支持在后台或锁屏状态下快速管理下载任务。
- [ ] **文件校验逻辑**：续传完成后强制进行文件大小及 Hash 校验，防止因分片拼接导致的安装包或产物损坏。
